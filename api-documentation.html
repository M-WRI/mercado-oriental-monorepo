Mercado Oriental API

REST API for the Mercado Oriental vendor administration platform. Manage shops, products, variants, attributes, and view business analytics.

Base URL  http://localhost:8000  ·  All endpoints are prefixed with /api.

Authentication

The API uses JWT bearer tokens. Obtain a token via the /api/auth/login or /api/auth/register endpoints, then include it in every request to protected routes.

Header format

Authorization: Bearer <token>

Tokens expire after 7 days. When a token is missing or invalid the API responds with 401 and an error body identifying the issue.

All endpoints except POST /api/auth/register and POST /api/auth/login require a valid bearer token.

Error handling

Every error response follows the same shape, making it easy to build generic error handling on the client.

Error response format

{
  "case": "product_name",
  "code": "MISSING"
}

The case field identifies what went wrong (which field or resource), while code categorises how it went wrong. This pair maps directly to i18n translation keys on the frontend: errors.{case}.{code}.

Error codes

CodeHTTP statusMeaning
MISSING400A required field was not provided
INVALID400A field value is malformed or out of range
NOT_FOUND404The requested resource does not exist
DUPLICATE409A unique constraint was violated
UNAUTHORIZED401No token or token expired
FORBIDDEN403Authenticated but not allowed
SERVER_ERROR500Unexpected internal error
